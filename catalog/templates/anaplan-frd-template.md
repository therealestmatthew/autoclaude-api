# FRD Template — Anaplan

The Functional Requirements Document captures what the Anaplan solution must do for the business, written before model design starts.

## Sections

- Business context, process scope, in-scope LOBs
- Stakeholder map and decision rights
- Functional requirements (use-case shaped, with priority)
- Data inputs / outputs
- Reporting and visualization needs
- Integration touchpoints
- Out-of-scope items (explicit)
- Acceptance criteria per requirement

## How we use it

- Author this *before* writing a Model Design Spec — design without locked requirements is the #1 cause of rebuild work
- Reviewed by the client business lead and signed off before architecture
- Sourced from discovery workshops + existing process docs
