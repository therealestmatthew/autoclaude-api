# Status reports

One file per weekly status: `YYYY-MM-DD.md`. Use the template below.

```markdown
# Status — {{client}} — {{date}}

## Outcomes this week

(What materially changed for the client. Outcome-shaped, not output-shaped.)

## Output this week

(Concrete deliverables shipped this week.)

## Next week

(What we're committing to next week. 3–5 items max.)

## Risks / blockers

(New or escalating risks. Reference scoping.md risks; flag new ones.)

## Decisions needed from client

(Specific, named decisions and when we need them by.)

## Hours / burn

(If relevant: hours this week, hours-to-date, % of budget.)
```
